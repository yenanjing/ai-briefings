## 📚 arXiv 精选论文

### 9🌟 Rethinking Language Model Scaling under Transferable Hypersphere Optimization
- **机构**: Microsoft
- **作者**: Liliang Ren, Yang Liu, Yelong Shen, Weizhu Chen
- **arXiv**: [2603.28743](https://arxiv.org/abs/2603.28743)
- **代码**: [github.com/microsoft/ArchScale](https://github.com/microsoft/ArchScale)
- **分类**: cs.LG
- **核心创新**: 提出 HyperP（Hypersphere Parameterization），首个在 Frobenius-sphere 约束下跨模型宽度、深度、训练 token 数和 MoE 粒度转移最优学习率的框架。证明 weight decay 在 Muon 优化器的 Frobenius sphere 上是一阶空操作，单次小规模调参结果可直接转移至所有计算预算，实现 1.58× 计算效率提升。同时提出 SqrtGate 门控机制，改善 MoE 跨粒度的稳定性。
- **评分理由**: Microsoft 顶级团队，解决 LLM 超参转移这一工程核心痛点，理论严谨，有开源代码加分。

### 8🌟 See it to Place it: Evolving Macro Placements with Vision-Language Models
- **机构**: Harvard University（Kempner Institute）/ Google DeepMind
- **作者**: Ikechukwu Uchendu, Swati Goel, Karly Hou, Ebrahim Songhori, Kuang-Huei Lee, Joe Wenjie Jiang, Vijay Janapa Reddi, Vincent Zhuang
- **arXiv**: [2603.28733](https://arxiv.org/abs/2603.28733)
- **代码**: 暂无开源代码
- **分类**: cs.LG
- **核心创新**: 提出 VeoPlace（Visual Evolutionary Optimization Placement），将 VLM 视觉推理能力引入芯片宏单元布局优化。VLM 无需微调，直接作为高层规划器约束低层放置算法的搜索空间，通过进化过程迭代优化。在 9/10 公开基准上超越最优学习方法，线长减少最高达 32%；对解析式布局器同样有效，DREAMPlace 性能提升最高 4.3%。
- **评分理由**: Google DeepMind + Harvard 联合团队，将 VLM 迁移到电子设计自动化（EDA）新领域，实验充分，结果显著。

### 8🌟 Stepwise Credit Assignment for GRPO on Flow-Matching Models
- **机构**: Carnegie Mellon University / Adobe Research
- **作者**: Yash Savani, Branislav Kveton, Yuchen Liu, Yilin Wang, Jing Shi, Subhojyoti Mukherjee, Nikos Vlassis, Krishna Kumar Singh
- **arXiv**: [2603.28718](https://arxiv.org/abs/2603.28718)
- **代码**: [stepwiseflowgrpo.com](https://stepwiseflowgrpo.com)
- **分类**: cs.LG, cs.AI, cs.CV
- **核心创新**: 指出 Flow-GRPO 对所有去噪步骤使用均一奖励的根本缺陷：忽略了不同步骤决定构图（低频）与细节（高频）的时间结构差异。提出 Stepwise-Flow-GRPO，利用 Tweedie 公式获取中间奖励估计并引入逐步增益优势，实现更快收敛和更高最终奖励，并提出 DDIM 启发的 SDE 改善奖励质量。
- **评分理由**: CMU + Adobe Research 强强联合，解决 Flow 模型 RL 训练中的核心信号分配问题，有项目主页，开源代码加分。

### 7🌟 HandX: Scaling Bimanual Motion and Interaction Generation
- **机构**: University of Illinois Urbana-Champaign / Specs Inc. / Snap Inc.
- **作者**: Zimu Zhang, Yucheng Zhang, Xiyan Xu, Ziyin Wang, Sirui Xu, Kai Zhou, Bing Zhou, Chuan Guo, Jian Wang, Yu-Xiong Wang, Liang-Yan Gui
- **arXiv**: [2603.28766](https://arxiv.org/abs/2603.28766)
- **代码**: [handx-project.github.io](https://handx-project.github.io)
- **分类**: cs.CV
- **核心创新**: 构建大规模双手动作与交互数据集 HandX，包含 54.2 小时、5.9M 帧、490K 文本的精细化运动捕捉数据，支持 Triplet 结构多粒度标注。基准测试 diffusion-based 和 autoregressive 两种生成范式，验证数据量和模型容量均有明确 scaling trend。
- **评分理由**: UIUC 领衔，填补了双手精细运动生成数据与基准的空白，数据贡献显著，有项目主页，开源代码加分。

### 7🌟 AdaptToken: Entropy-based Adaptive Token Selection for MLLM Long Video Understanding
- **机构**: Microsoft Spatial AI Lab / EPFL / ETH Zurich
- **作者**: Haozhe Qi, Kevin Qu, Mahdi Rad, Rui Wang, Alexander Mathis, Marc Pollefeys
- **arXiv**: [2603.28696](https://arxiv.org/abs/2603.28696)
- **代码**: [haozheqi.github.io/adapt-token](https://haozheqi.github.io/adapt-token)
- **分类**: cs.CV, cs.AI
- **核心创新**: 提出 AdaptToken，基于 MLLM 自身响应熵作为全局控制信号，跨视频片段比较相关性并在熵足够低时提前停止处理。在 VideoMME、LongVideoBench 等四个长视频基准上稳定提升（平均 +6.7 vs Qwen2.5-VL 7B），AdaptToken-Lite 版本将推理时间减半且性能相当。
- **评分理由**: Microsoft Spatial AI Lab + EPFL + ETH Zurich 强联合，解决长视频 token 效率实际痛点，有项目主页，开源代码加分。

### 7🌟 Stop Probing, Start Coding: Why Linear Probes and Sparse Autoencoders Fail at Compositional Generalisation
- **机构**: Mila – Québec AI Institute / Université de Montréal / Santa Clara University / Cold Spring Harbor Laboratory
- **作者**: Vitória Barin Pacela, Shruti Joshi, Isabela Camacho, Simon Lacoste-Julien, David Klindt
- **arXiv**: [2603.28744](https://arxiv.org/abs/2603.28744)
- **代码**: 暂无开源代码
- **分类**: cs.LG
- **核心创新**: 从理论与实验两方面证明线性探针和稀疏自编码器（SAE）在组合泛化任务上的系统性失败根源在于字典学习——而非推断过程。SAE 学习的字典方向存在根本性偏差，即使替换为同样字典的逐样本 FISTA 也无法弥合差距。将问题定性为可扩展字典学习挑战，为可解释性研究重新定向。
- **评分理由**: Mila + Simon Lacoste-Julien，可解释性领域重要理论贡献，清晰识别现有方法的绑定约束，对整个领域有指导意义。

### 7🌟 SOLE-R1: Video-Language Reasoning as the Sole Reward for On-Robot Reinforcement Learning
- **机构**: Northeastern University / Boston Dynamics AI Institute（推测）
- **作者**: Philip Schroeder, Thomas Weng, Karl Schmeckpeper, Eric Rosen, Stephen Hart, Ondrej Biza
- **arXiv**: [2603.28730](https://arxiv.org/abs/2603.28730)
- **代码**: 暂无开源代码
- **分类**: cs.RO, cs.CL, cs.CV
- **核心创新**: 提出 SOLE-R1，仅用视频-语言模型推理作为唯一奖励信号进行在线机器人强化学习，无需人工设计奖励函数或稠密状态信息。VLM 通过观察机器人视频并比照任务描述进行 reasoning，直接输出稀疏奖励，在真实机器人任务上验证可行性。
- **评分理由**: 将 VLM reasoning 直接闭环到机器人 RL 训练中，方向前沿；机构信息不完全明确，评分保守。

### 6🌟 Dynamic Dual-Granularity Skill Bank for Agentic RL
- **机构**: 中国科学院自动化研究所（CASIA）
- **作者**: Songjun Tu, Chengdong Xu, Qichao Zhang, Yaocheng Zhang, Xiangyuan Lan, Linjing Li, Dongbin Zhao
- **arXiv**: [2603.28716](https://arxiv.org/abs/2603.28716)
- **代码**: [github.com/TU2021/D2Skill-AgenticRL](https://github.com/TU2021/D2Skill-AgenticRL)
- **分类**: cs.AI
- **核心创新**: 提出 D2Skill，在 Agentic RL 中同时以轨迹级和步骤级组织可复用经验，构建双粒度 Skill Bank。低层步骤级技能捕捉精细操作模式，高层轨迹级技能提供宏观规划指导，有效提升 agent 在复杂任务中的样本效率和泛化能力。
- **评分理由**: 中科院自动化所，Agentic RL 技能复用方向，有开源代码加分；创新程度中等。

### 6🌟 BACE: LLM-based Code Generation through Bayesian Anchored Co-Evolution
- **机构**: WSO2（Santa Clara, CA）
- **作者**: Kaushitha Silva, Srinath Perera
- **arXiv**: [2603.28653](https://arxiv.org/abs/2603.28653)
- **代码**: 暂无开源代码
- **分类**: cs.NE, cs.SE
- **核心创新**: 提出 BACE，将 LLM 代码生成建模为代码种群与测试种群的贝叶斯锚定协同进化过程。两个种群相互影响，代码影响测试方向，测试反向筛选代码质量，通过协同进化优于传统序列化改进方法。收录于 GECCO '26。
- **评分理由**: 工业界创新应用，遗传算法与 LLM 结合思路新颖，已录用正式会议；机构非顶级学术机构，评分中等。

### 6🌟 Pandora: Articulated 3D Scene Graphs from Egocentric Vision
- **机构**: MIT SPARK Lab
- **作者**: Alan Yu, Yun Chang, Christopher Xie, Luca Carlone
- **arXiv**: [2603.28732](https://arxiv.org/abs/2603.28732)
- **代码**: 暂无开源代码
- **分类**: cs.RO, cs.CV

### 5🌟 On-the-fly Repulsion in the Contextual Space for Rich Diversity in Diffusion Transformers

- **评分理由**: Tel Aviv University（Daniel Cohen-Or 为 CV/图形学领域权威），无训练改进思路简洁，但影响范围有限。
